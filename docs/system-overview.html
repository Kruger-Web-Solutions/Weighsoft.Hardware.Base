Architecture

System Overview

Four layers — hardware interface, firmware services, communication channels, and web UI — working together to deliver real-time, legally-traceable weighing.

Layer 1

Hardware Interface

The ADS1231 is a 24-bit sigma-delta ADC connected to the ESP32 via bit-banged GPIO. Four pins control the entire measurement pipeline.

GPIO 25DOUT — Data ready + serial data (INPUT)
GPIO 26SCLK — Serial clock, bit-banged (OUTPUT)
GPIO 27PDWN — Power down, HIGH = active (OUTPUT)
GPIO 33SPEED — 10 SPS (LOW) / 80 SPS (HIGH) (OUTPUT)

Layer 2

Firmware Architecture

Services compose reusable framework components. Each application service owns its own channels inline — no separate protocol-settings services needed.

Build Targets

PlatformIO Branches

Three hardware targets share the same firmware codebase. Feature flags and partition tables differ per target.

🎯

Target: node32s

Primary production target. Custom partition table splits flash into 2× 1.5 MB OTA slots + 960 KB SPIFFS for web assets and config.

platformio.ini [env:node32s]
🗂️

Partition Table

otadata 8KB · factory 1.5MB · ota_1 1.5MB · spiffs 960KB. Dual OTA allows safe updates — one slot active, one receives new firmware.

partitions_custom.csv
⚡

Performance

240 MHz dual-core Xtensa LX6. 520 KB SRAM. Arduino-ESP32 framework. All async I/O via ESPAsyncWebServer prevents measurement blocking.

framework = arduino
🔌

Target: esp32dev

Generic ESP32 dev board — useful for rapid prototyping. Same firmware, default partition layout (no custom CSV).

📦

Library Stack

ArduinoJson 6.x · ESPAsyncWebServer · AsyncMqttClient 0.9.x. All shared with node32s target.

⚙️

Feature Flags

FT_PROJECT, FT_SECURITY, FT_MQTT, FT_NTP, FT_OTA enabled. FT_BLE disabled by default.

📡

Target: esp12e (ESP8266)

ESP-12E module at 160 MHz. Single-core — no mutex support; WiFi can briefly block the measurement loop.

platform = espressif8266
⚠️

Constraints

80 KB usable heap. No BLE. GPIO logic for LED is inverted vs ESP32 (LOW = ON). Reduced concurrent WebSocket clients.

🔗

Compatibility

Same REST/WS/MQTT/SPIFFS API. Factory settings can target either platform via #{platform} placeholder substitution.

Layer 4

React Frontend

React 18 + Material-UI 5 SPA served from SPIFFS or PROGMEM. Authenticated via JWT, communicates over REST and WebSocket.

🔐

Authentication

Login page issues JWT via /rest/signIn. Token stored in session, attached to all subsequent requests.

📡

WiFi Configuration

Network scan, station credentials, AP fallback mode. Changes write to LittleFS and trigger reconnection.

⚖️

Weighing Panel

Live weight display, tare/zero controls, calibration wizard, seal/unseal UI, unit selector. WebSocket-driven real-time updates.

📋

Audit Trail Viewer

Admin-only table of all OIML events. Timestamps, event types, old/new values, event counter column.

🔧

MQTT / NTP Settings

Runtime broker configuration, topic prefix, NTP server and timezone. All persisted to flash.

🚀

OTA Updates

Upload firmware binary or trigger update. New binary lands in ota_1 slot; reboot activates it.

Compliance

OIML D31:2023 Compliance

Five requirements implemented to support legal-for-trade type approval: event counter, audit trail, electronic seal, stability, and traceability.

Weighing Panel Tabs

Live Weight

Monospace large display updates ~2Hz via WebSocket. Shows gross or net depending on mode. Unit displayed alongside.

Status Badges

STABLE (green) when rolling 8-sample range < minDivision. Center-of-zero badge when |gross| ≤ 0.25×div. ADC-disconnected / Overload / Underload alerts.

Commands

Tare sends {tare:true} via REST POST. Preset Tare opens dialog for preset_tare value. Zero is disabled when instrument is sealed.

Seal Lock

When seal_active=true, an orange Alert banner appears and all inputs are disabled. Cannot save changes until unsealed.

Measurement Unit

Dropdown: kg / g / lb. Triggers recalculation of all display values.

Scale Parameters

Max Capacity · Min Division (d) · Decimal Places (0–6) · Samples to Average (1–32) · Speed Mode radio (10 SPS vs 80 SPS).

Zero Tracking

Checkbox enables AZT. When enabled, shows Zero Tracking Range field (max correction per cycle). Also sets Power-On Zero Range (%) and Manual Zero Range (%).

Event Counter

Read-only display. Monotonic uint32. Never editable — shown to demonstrate OIML compliance to an inspector.

useRest hook

Calls GET /rest/weighing on mount. "Apply Configuration" calls POST /rest/weighing with changed fields. Shows success/error snackbar.

Audit Table

Read-only table via GET /rest/weighingAudit. Columns: seq, timestamp, event, parameter, old_value, new_value, event_counter.

Event Colors

seal and unseal_fail rows show in red. unseal rows show in green. All other events in default color.

Admin Only

Route protected by RequireAdmin. Only users with admin:true in JWT can access this tab.

NDJSON Storage

Stored in /audit/weighing.json on LittleFS. FIFO, max 100 entries. Lives on a separate flash partition — survives OTA firmware updates.

Polling Mode

Uses setInterval at 2-second cadence. Good for debugging or low-bandwidth environments where WebSocket is impractical.

Display

Shows weight, gross/net/tare grid, mode chip, stability chip, ADC connected indicator, raw ADC value, event counter, and overload/underload chips.

vs Live Stream

REST tab is read-only (no buttons). Live Stream tab has controls. REST is slower (2s) but works through firewalls that block WebSocket upgrades.

BLE Service

Service UUID: 56780000-e8f2-537e-4f6c-d104768a5678

Characteristic

UUID: 56780001-…5678 with READ, WRITE, NOTIFY properties. Same JSON schema as REST/WS.

Commands via BLE

Write {tare:true}, {preset_tare:1.5}, {zero:true}, {calibrate:true,known_weight:5.0}, {seal:true}, {unseal:true,pin:"800125"}.

Status

FT_BLE currently disabled. Phase 2 feature. UI tab shows documentation only. BLE settings page still accessible to configure device_name.

Theme System

Initial Mode

Detected via window.matchMedia('prefers-color-scheme: dark'). Matches system preference on first visit.

Per-User Persistence

On login, fetches GET /rest/userTheme. On toggle, calls PUT /rest/userTheme. Each user account stores their preferred theme.

🎨

Brand Colors

Primary: Curious Blue #2596D1. Secondary: Bunting #2B3449. Responsive fonts via MUI responsiveFontSizes().

🔄

Optimistic Toggle

UI switches immediately on click. toggleTheme() updates state first, then persists to backend async. Reverts on error.
